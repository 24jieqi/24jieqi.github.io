    一、useState

    1.1 定义

    让函数组件中拥有 state 变量。它接收一个初始化的 state，返回是一个数组，数组里有两个元素，第一个元素是当前状态值和另一个更新该值的方法。 ​

    1.2 原理

    返回是一个数组，利用解构赋值简化代码

      //原始方式
  const Message= () => {
  const messageState = useState("");
  const message = messageState[0]; // 是一个空字符串
  const setMessage = messageState[1]; // 是一个方法
}

 //解构赋值
  const Message = () => {
  const [message, setMessage] = useState("");
}

    ​

    ​

    1.3 使用注意点

    1、useState Hook 一次只能声明一个状态变量，这个状态变量可以是任意类型的。 2、第一个参数 state 只能读，不能对它进行修改，例如：state++，state.splice 等操作。 ​

    ​

    1.4 使用场景

    例 1：

    //设定状态作为输入框的值
const [addInputValue,setAddInputValue]=useState('');
<input type="text" value={addInputValue} onChange={(e)=>{setAddInputValue(e.target.value)}} placeholder="enter your todo"/>

    四、usememo

    4.1 语法格式

    const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);

    4.2 语法说明

    它接受两个参数：

    • 第一个参数为回调函数（计算过程，必须返回一个结果）
    • 第二个参数是依赖项（数组），当依赖项中某一个发生变化，结果将会重新计算

    4.3 作用

    ​

    作用： 把“创建”函数和依赖项数组作为参数传入 useMemo，它仅会在某个依赖项改变时才重新计算 memoized 值。这种优化有助于避免在每次渲染时都进行高开销的计算。 ​

    4.4 使用注意点

    注意：

    • 传入 useMemo 的函数会在渲染期间执行。请不要在这个函数内部执行与渲染无关的操作，诸如副作用这类的操作属于 useEffect 的适用范畴，而不是 useMemo。
    • 如果没有提供依赖项数组，useMemo 在每次渲染时都会计算新的值。
    • 因为是缓存计算结果，所以参数 1 回调函数里面一定要有 return 返回值

    4.5 案例

    说明案例：通过 usememo 对 Calsum 的返回值进行缓存，达到只有 count 转态的值改变时，才会去执行 Calsum 函数。

        const [count, setcount] = useState(0);
    const [otherCount, setotherCount] = useState(0);

    const addCount = () => {
        setcount(count + 1);
    };

    const addotherCount = () => {
        setotherCount(otherCount + 1);
    }

    const Calsum = (count) => {
        console.log("计算了count");
        let _sum = 0;
        for (let i = 0; i < count; i++) {
            _sum += count;
        }
        return _sum;
    }

    // let sum = Calsum(count);
    const sum = useMemo(() => { return Calsum(count) }, [count])

    return (
        <div>
            {sum}
            <div><button onClick={addCount}>+count</button>{count}</div>
            <div><button onClick={addotherCount}>+otherCount</button>{otherCount}</div>
        </div>
    )

    五、usecallback

    5.1 语法格式

    const memoizedCallback = useCallback(
  () => {
    doSomething(a, b);
  },
  [a, b],
);

    5.2 语法说明

    • 传递内联回调和依赖项数组。useCallback 将返回回调的记忆版本，该版本仅在依赖项之一发生更改时才会更改。这在将回调传递给依赖引用相等性的优化子组件时很有用，以防止不必要的渲染。

    六、useEffect

    6.1 语法格式

    useEffect(callback,deps)//（回调，依赖数组）

    ​

    6.2 语法说明

    第一个参数: 在组件发生 componentDidMount(组件加载结束) ， componentDidUpdate(状态变化) 和 componentWillUnmount(组件卸载)情况下都会触发 useEffect(函数参数)中函数参数的执行

    第二个参数：

    1. 第二个参数是数组，数组中元素是状态变量， 当状态变量在其中才表示该状态被监听，没有在其中则不被监听.
    2. 如果没有第二个参数，则默认所有状态都会被监。
    3. 如果第二个参数是没有元素的数组，所有状态都不会被监听

    作用: Effect Hook 可以让你在函数组件中执行副作用操作

    • 副作用:数据获取、设置/销毁定时器以及手动更改 React 组件中的 DOM 都属于副作用
    • 在函数组件中，每当 DOM 完成- -次渲染， 都会有对应的副作用执行

    6.3 使用案例

    案例 1：只有一个回调的参数

     useEffect(() => {
	console.log("useEffect()")
  })

    ​

    案例 2：第二个参数依赖数组为空

      const [num,setnum] = useState(0)

  useEffect(()=>{
        setInterval(()=>{
            console.log("执行effect");
        },1000)
    },[])

    return (
        <div>

        //这里依赖数组为空，可以实现点击button不会造成useeffect副作用函数的执行

    案例 3：返回值销毁函数

    一个实用的清除示例----定时器

    import React,{useState, useEffect} from 'react';

function App() {
	const[name, setName] = useState<string>("小王");
	const[age, setAge] = useState<number>(20);

	function forEffect() {

		console.log("加载了App17");

		let id = setInterval(()=>{
			console.log("小王");
		}, 1000);

		//从第二次执行useEffect开始，后面执行都先执行return返回的函数
		return ()=>{
			console.log("清楚");
			clearInterval(id);
		}
	}

	useEffect(forEffect, [name, age]);

	return(
		<div>
			{name},{age}
			<button onClick={()=>{setName("小王"+ Math.floor(Math.random()*100))  }}>改变状态name</button>
			<button onClick={()=>{setAge(Math.floor(Math.random()*100)) }}>改变状态age</button>
		</div>
	);
}

export default App;

    6.4 useEffect 在组件中执行的顺序

    ​

    每次副作用执行，都会返回个新的销毁函数 销毁函数会在下一个副作用逻辑之前执行(DOM 渲 染之后) 组件销毁也会执行销毁函数 ​

    十、自定义 hooks

    ** 定义:** 除了使用内置的 Hooks 之外，还可以自定义 Hooks，自定义 Hooks 针对不同组件实现不同状态逻辑复用。 ​

    使用规范： ​

    • 自定义 Hooks 是一个函数，约定函数名称必须以 use 开头，React 就是通过函数名称是否以 use 开头来判断是不是 Hooks。

    • Hooks 只能在函数组件中或其他自定义 Hooks 中使用，否则，会报错！

    • 自定义 Hooks 用来提取组件的状态逻辑，根据不同功能可以有不同的参数和返回值（就像使用普通函数一样）

    ​

    使用场景: 将组件状态逻辑提取到可重用的函数（自定义 Hooks）中，实现状态逻辑复用。 对业务，功能进行抽离的时候。 ​

    2、防抖

    2.1 需要改进的地方

    第一次触发时，不进行防抖处理 ​

    2.2 实现思路

    ​

    1. 至少需要传入两个，事件处理函数，和需要延迟的时间
    2. 保存定时器，当第一次触发时执行定时器
    3. 在延迟时间内再次触发则清楚定时器

    ​

    2.3 useDebounce 实现代码

    const useDebounce = (fun: Function, delay: number) => {
  const time1 = useRef<any>(null);
  const first = useRef<boolean>(true);
  return useCallback(() => {
    if (time1.current) {
      clearTimeout(time1.current);
    }
    if (first.current) {
      fun();
      first.current = false;
    } else {
      time1.current = setTimeout(() => {
        fun();
      }, delay)
    }


  }, [fun, delay])
}

    3、节流

    3.1 需要改进的地方

    第一次触发时，不进行节流处理 ​

    3.2 实现思路

    ​

    1. 至少需要传入两个，事件处理函数，和需要延迟的时间
    2. 保存定时器，当第一次触发时执行定时器
    3. 在延迟时间内再次触发则直接返回，不做处理

    ​

    3.3 useThrottle 实现代码

    const useThrottle = (fun: Function, delay: number) => {
  const time1 = useRef<any>(null);
  const throttle = useCallback(() => {
    if (time1.current) {
      return;
    }
    time1.current = setTimeout(() => {
      fun();
      time1.current = null;
    }, delay)
  }, [delay, fun])
  return throttle

    4、屏幕高度 hooks

    4.1 需要改进的地方

    结合节流去获取屏幕高度

    4.2 实现思路

    1. 返回值为当前屏幕高度
    2. 当调用 useScreenHight 时就开始进行监听屏幕高度
    3. 屏幕高度的改变用一个状态去保存

    4.3 useScreenHight 实现代码

    const useScreenheight = () => {
    const [height, setheight] = useState(document.documentElement.clientHeight);

    const handelSize = useCallback(() => {
        setheight(document.documentElement.clientHeight);
    }, []);

    useEffect(() => {
        window.addEventListener("resize", handelSize);
        // 为了防止一直监听所以在方法移除时，使用return的方式移除监听
        return () => {
            window.removeEventListener("resize", handelSize)
        }
    }, [])
    return height;
}

    5、倒计时

    5.1 需要改进的地方

    对倒计时应该有，开始，暂停，重新开始计时的功能 倒计时为 0 时，可以进行有一些操作

    5.2 实现思路

    1. 传入倒计时总长度
    2. 定义一个倒计时函数
    3. 把当前时间和回调函数传出
    4. 调用 useDownCount 是可以解构到时间和触发倒计时的函数

    5.3 useDownCount 实现代码

    const useCountdown = (initcount: number) => {
  const [count, setcount] = useState(initcount);
  let time1: any = null;
  const handleCount = () => {
    setcount(initcount)
    time1 = setInterval(() => {
      setcount((count) => count - 1)
    }, 1000)
  }
  useEffect(() => {
    return () => {
      clearInterval(time1)
    }
  }, [])

  return {
    count,
    handleCount
  }
}
